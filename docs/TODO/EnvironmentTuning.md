# [~] EnvironmentTuning — подсказки пользователю по тюнингу окружения

Трекер требований/рекомендаций к окружению (терминал, мультиплексор, SSH), от которых
зависит корректная работа редактора. Цель — научиться **детектировать** такие
ситуации и **подсказывать** пользователю, что подкрутить, а не «молча чинить» за него
(тем более не ценой регрессий вроде утечки кодировки в соседние панели tmux).

Статусы: `[ ]` — открыта, `[~]` — в работе, `[x]` — сделана.

---

## Мотивация

Часть функций редактора работает корректно только при определённых настройках
окружения, которые пользователь не обязан знать. Пример — Ctrl+Tab в tmux (см. ниже):
клавиша физически не доходит до приложения, пока в tmux не включён `extended-keys`.
Мы намеренно не подкручиваем чужой tmux из кода, поэтому нужен механизм, который
честно объяснит пользователю причину и даст рецепт.

---

## Открытый вопрос: как доносить до пользователя

Способ подачи ещё **не выбран**. Кандидаты (идеи, без обязательств):

- намёк/индикатор в статусбаре, когда обнаружено «недонастроенное» окружение;
- one-shot нотификация при старте (с возможностью «больше не показывать»);
- команда-«doctor» — отчёт по окружению со списком рекомендаций;
- запись рекомендаций в лог.

Источник сигналов у нас уже есть: `TerminalEnvironmentService`
(`src/Controllers/TerminalEnvironment/TerminalEnvironmentService.ts`) детектит
окружение и capability `extended-keys` (env-хинты + fire-and-forget проба
клавиатурного протокола). Его естественно переиспользовать как вход для подсказок.

## Список рекомендаций

Формат пункта: симптом → причина → что подкрутить.

### [ ] tmux: Ctrl+Tab / Ctrl+Shift+Tab не переключают вкладки (вставляется Tab)

- **Симптом.** Внутри tmux Ctrl+Tab вместо переключения на следующую вкладку
  вставляет в файл символ табуляции.
- **Причина.** Tab — это байт `0x09`; модификатор Ctrl к нему добавляется только через
  kitty keyboard protocol / modifyOtherKeys. Включение этих режимов мы шлём
  попанельно (`writeDirect`, `src/Backend/NodeTerminalBackend.ts:244`), чтобы не течь
  в соседние панели. Но tmux отдаёт расширенные клавиши приложению **только** при
  `extended-keys on`, а дефолт tmux (≥ 3.2) — `off`. При `off` tmux игнорирует
  `\x1b[>15u` и отдаёт Ctrl+Tab как обычный `0x09`.
- **Что подкрутить (пользователю).** В `~/.tmux.conf`:
  ```tmux
  set -s extended-keys on
  # при необходимости — сообщить tmux, что внешний терминал умеет extended keys:
  set -as terminal-features ',*:extkeys'
  ```
  После этого попанельное включение kitty заработает без утечки кодировки в другие
  панели (Ctrl+S как префикс tmux продолжает работать).
- **Фолбэк уже сейчас.** `Ctrl+PageDown`/`Alt+PageDown` и `Ctrl+PageUp`/`Alt+PageUp`
  (`src/Controllers/Actions/TabActions.ts:9,24`) переключают вкладки и **без**
  `extended-keys` — модификаторы на спец-клавишах кодируются стандартным
  `;mod`-параметром, который tmux пропускает всегда.
- **Почему не чиним кодом.** Возврат tmux DCS passthrough для input-режимов — это
  прошлый баг (утечка кодировки во все панели, ломался Ctrl+S-префикс; коммит
  `3d20e6f`). Авто-`tmux set -s extended-keys on` из кода мутирует серверную опцию
  чужого tmux — отвергнуто в пользу подсказки.
