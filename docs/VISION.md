# Vision — позиционирование Vexx

Этот документ — не про архитектуру и не про фичи. Он про то, **чем Vexx является для внешнего
мира** и куда осмысленно двигаться. Цели и неизменяемые ограничения реализации — в
[GOAL.md](../GOAL.md); здесь — стратегия, которая пересматривается по мере того, как меняются
факты.

Документ живой. Если факт из раздела «Ограничения реальности» перестал быть фактом — направление
пересматривается.

---

## Главная мысль

**VS Code кончается на ssh. Vexx — нет.**

Не лучше, не проще, не быстрее. Просто **есть там, где VS Code нет**, в момент, когда человека туда
занесло. Спрос создаёт не качество, а география.

---

## Активы

- **VS Code-совместимый extension host вне Electron и браузера.** Расширения VS Code, запущенные
  там, где нет ни Electron, ни браузера. Аналогов, по сути, нет.
- **TUI-движок на TypeScript, который не Ink.** Grid + double buffering + диффы ANSI + layout +
  Kitty keyboard protocol. Оплачен запретом на фреймворки из GOAL.md — и этот запрет окупается в
  цифрах старта (см. «Скорость»).
- **LSP.** Работает — значит это IDE, а не «редактор с подсветкой».
- **Собственно редактор** — немодальный, с мышечной памятью VS Code.

**Ранжировать их — ошибка.** Редактор — слабейший актив как вещь, которую *продают*, и сильнейший
как вещь, которую *принимают*: захотеть можно только редактор, никто не просыпается с желанием
поставить MCP-сервер. А extension host — не отдельная ставка, а **механизм**, которым редактор
получает батарейки: языковая поддержка VS Code живёт в расширениях, мы их запускаем — значит языки
у нас есть даром.

Один актив, две выплаты: **для человека — языки из коробки, для агента — языковой мозг через MCP.**

## Аудитория — кто именно приходит

Терминал растёт не сам по себе. Он растёт **потому что туда загнали агенты**: Ghostty, Warp, tmux,
девконтейнеры — фон, а двигатель — Claude Code и его родня.

Ключевое: аудитория, которая сейчас прибывает в терминал, — это **не терминальные люди**.
Терминальные люди никуда не уходили и давно поделены вимом. Прибывают вэсэкодовцы, которые раньше
заходили в терминал за `git push`, а теперь сидят там часами, потому что там живёт агент.

**Эмоция — не недовольство, а вытеснение.** Люди не в поиске лучшего редактора. Они VS Code
**любят** и им довольны (автор — тоже; «терпят VS Code» — ложная посылка). Их не тянет уходить — их
**выпихивает**: ssh, контейнер, залоченный бастион, агент в терминале. Настроение не «ненавижу
VS Code, дай лучше», а **«люблю VS Code и сейчас не могу им пользоваться, дай ближайшее»**.

Это выгоднее, чем недовольство: мы **обслуживаем их любовь к VS Code, а не воюем с ней**. Мышечная
память, клавиши, `settings.json`, темы — это не «клонирование», а точность воспроизведения, ровно
то, за что благодарят.

**Люди вообще не меняют редакторы.** Не потому что новый плох — просто не меняют (сверстники автора
сидят в IDEA и не собираются никуда, хотя могли бы; сам автор переехал только через переходный
конфиг, который потом рассосался). Значит **собирать недовольство неоткуда** — ни редактором, ни
настройкой, ни привычками. Единственный вход — присутствие в момент вытеснения.

**Наша ниша — не «терминальный редактор»** (статично, занято вимом), **а «редактор для тех, кого
агенты затащили в терминал»**. Потолок аудитории задаётся не тем, скольким людям Vexx понравится, а
тем, **сколько часов разработчики проводят вытесненными в терминал.** Это растёт, но это слайс от
чужой жизни, а не рынок редакторов. Денег мы и не берём — просто цифра в голове должна быть честная.

## Три работы — три адресата

Их надо держать раздельно, иначе стратегия слипается:

1. **Присутствие создаёт спрос.** Ты нужен, потому что там ничего нет. Не «лучше VS Code» — а «есть,
   где его нет».
2. **Знакомость выигрывает выбор.** Когда человек уже вытеснен, он выбирает между Vexx, Helix и vim.
   Вот здесь работают немодальность и клавиши VS Code — **против Helix**, а не против VS Code.
3. **Батарейки — порог доверия.** Языки не работают → ты игрушка, а не замена. Это допуск на поле, а
   не аргумент. И это **компенсация**, а не философия: VS Code и так «легко настраивается» (профиль
   натыкивается мышкой за минуту, никто с ним не борется), так что «лёгкая настройка» против него —
   ноль. Мы обязаны вшить батарейки не ради красоты, а потому что marketplace закрыт и профиль
   мышкой натыкать **нельзя**.

Единственное реальное трение у VS Code — тоже географическое: **не с редактором, а с его доставкой**
в нужное место. Remote-SSH тяжёлый и требует поднять серверную половину на той стороне (память,
залоченные боксы); code-server требует браузер и проброс порта. Вот и вся «борьба», что существует в
природе. Мы её снимаем присутствием.

## Прецедент: Helix

Единственный релевантный пример — соло-старт, без компании и денег, дошедший до массовости. Блаж
Храстник выложил Helix на HN 17 мая 2021; пять лет спустя — 38k+ звёзд, 195 контрибьюторов за
релизный цикл, 1.0 так и не случилось. Ни раунда, ни ускорителей.

**Что переносится (плейбук):**
- **Одна фраза про боль, а не про фичи.** Топ-комментарий на HN был не про возможности: *«the first
  editor I have not had to fight with on day one».*
- **Дефолты — это и есть продукт.** У Helix за пять лет **так и не появилась система плагинов** — и
  это заставило дефолты быть отличными и убрало запасной выход «пойди донастрой». Пустота была фичей.
- **Работающий редактор за 30 секунд.** Сделка явная и выполненная сразу (у нас есть SEA → один
  бинарь, `curl | sh`/brew).
- **Резкое мнение, из-за которого спорят.** Selection-first фехтовали — споры это рост. Наше
  «немодальный, клавиши VS Code, в терминале» тоже фехтуемо: вимеры полезут драться. Это хорошо.

**Что НЕ переносится (условия):**
- Helix ехал на племенной энергии r/rust; у нас её нет, и терминальная публика максимально скептична
  к Node по старту/памяти (см. «Скорость» — цифра должна быть в кармане).
- Аудитория Helix **уже существовала** (вимеры, уставшие от конфига). Наша — растущая гипотеза
  (вытесненные вэсэкодовцы).
- Двигатель Helix — история **перехода** («I switched from Neovim»). Наша аудитория довольна и
  вытеснена, поэтому наша история — не переход, а **«теперь могу»**: не «я ушёл с VS Code», а «я
  наконец получил своё там, где не мог». Пишется легче — в ней нет признания «я был неправ».

**Helix умирает — и это вакансия, а не приговор нам.** «Путь Helix, который в эпоху ИИ умрёт» —
верно ровно наполовину. Он умирает **от отсутствия того, что у нас работает**: в него структурно
нельзя вставить агента (плагинов нет), поэтому для Neovim есть avante/codecompanion, а для Helix
нет и не будет. Его величайшая сила — «расширять нечем, поэтому дефолты отличные» — в агентную эпоху
превращается в приговор. **Мы идём его путём, неся в руках именно ту вещь, отсутствие которой его
хоронит.**

И ещё Helix объясняет истинную роль extension host'а. Его батарейки оплачены **ручным
`languages.toml`, который поддерживают 195 человек.** Мы то же самое **берём из экосистемы, а не
содержим руками.** Extension host — не фича для пользователя, а **механизм дистрибуции конфигурации**,
который даёт батарейки для полусотни языков без комьюнити на 195 человек. Соло это единственный
способ — и поэтому он не должен светиться наружу как «ставьте расширения» (см. «Батарейки вшиваем»).

## Скорость — допуск на поле

Node в терминальном редакторе — красная тряпка. Но цифры переворачивают возражение в аргумент.

Замер (i7 12-го поколения, Balanced, батарея — консервативные условия):

| веха | median |
| --- | --- |
| пол Node (`node --version`) | 77 мс |
| первый кадр (workbench) | 87 мс |
| редактор готов (текст виден) | 103 мс |

**Заголовок здесь не 103, а 26.** 77мс из 103 — это пол самого рантайма; весь наш воркбенч стоит
**26мс поверх пола Node**, до первого кадра — 10мс. Это не оправдание за рантайм, это флекс: сколько
стоит движок. Оплачено запретом на Ink/React из GOAL.md — ограничение окупилось публично и в цифрах.
103мс к тому же **под порогом восприятия**, а по ssh старт вообще тонет в RTT.

**Рассказывать надо лестницу, а не число.** Текст с подсветкой за ~100мс; extension host
поднимается **лениво**, пока человек соображает; language server приезжает **асинхронно** дальше.
Ничто медленное не стоит на перцептивно-критическом пути — ровно как в самом VS Code. Одно число
читается как метрика; лестница читается как осознанный дизайн и делает 26мс правдоподобными.

**Что снять до поста** (та же машина, те же условия, таблицей в пост): vim · nvim голый ·
**nvim+LazyVim/NVChad** · helix · vexx. Ключевая строка — сконфигурированный Neovim: голый vim никто
не запускает, а реальная установка регулярно уезжает за сотню мс. Против голого vim мы проиграем
(Rust/C против рантайма) — и это честно называем.

**Зафиксировать в CI сейчас, пока цифра хорошая.** Есть `vitest.perf.config.ts` / `test:perf` —
повесить бюджет на старт. Число, которым хвастаемся в посте, обязано быть защищено тестом: extension
host, LSP, фичи будут откусывать по чуть-чуть, и 26 превратятся в 200 незаметно.

Скан дерева файлов **не** на критическом пути: синхронный walk и рекурсивный watcher убраны, индекс
строится чанкованным async-обходом с уступкой event loop (10k файлов ≈ 66мс фонового обхода, ничего
не блокирует), `node_modules`/`.git` исключены по умолчанию. Открытие большой монорепы на практике
быстрое. Остаток в [TODO/FileTreePerformance.md](TODO/FileTreePerformance.md) — полировка, не блокер.

## Ограничения реальности

Потолки, которые нам не сдвинуть. Стратегия обязана их обходить, а не бороться с ними.

- **Marketplace VS Code закрыт юридически.** ToS Microsoft запрещает сторонним продуктам ходить в
  marketplace — отсюда план с GitHub-топиком и OpenVSX (см. [TODO/Extensions.md](TODO/Extensions.md),
  Phase 10).
- **OpenVSX дырявый — но вопрос не «есть ли Pylance», а «хватает ли для вытесненного».** Из логики
  «терпят» следовало, что без коронных драгоценностей VS Code (Pylance, C#, Copilot, Remote-SSH —
  лицензированы только под VS Code) всё бессмысленно. Из логики вытеснения вопрос уже: **для языков,
  на которых аудитория реально работает в терминале, достаточно ли открытых серверов?** Для
  TypeScript/Go/Rust — да. Для Python — basedpyright + ruff, и это не эрзац. Дыра осталась, но
  сильно уже, чем в рамке «заменить весь VS Code».
- **Webview'ы в TUI невозможны в принципе.** Потолок совместимости — языковые и тулинговые
  расширения, не UI-heavy.
- **Функциональный паритет с VS Code недостижим** — асимптота, часть которой закрыта by design.
  Позиционирование на паритете строить нельзя.
- **Редакторы не монетизируются.** VS Code бесплатен, потому что Microsoft продаёт Azure и GitHub.
  Neovim и Helix бесплатны. Даже Zed **не продаёт редактор** — продаёт сервисы и AI (Pro $10/мес —
  хостед-кредиты к моделям). Продавать Vexx нельзя ни в каком виде.
- **`ide`-протокол Claude Code закрыт.** Официальные интеграции VS Code/JetBrains — через встроенный
  MCP-сервер `ide`: WebSocket на loopback, случайный порт, токен в `~/.claude/ide/<port>.lock`. Не
  документирован, не открыт третьим сторонам. Реверс = внутренний RPC, ломается на любом релизе.
- **Авторизация claude.ai запрещена третьим лицам.** Дословно из доков Agent SDK: *«Unless
  previously approved, Anthropic does not allow third party developers to offer claude.ai login or
  rate limits for their products, including agents built on the Claude Agent SDK.»* Значит
  **встроенный агент потребует отдельный `ANTHROPIC_API_KEY` и отдельный чек** при уже оплаченном
  Max. Для бесплатного редактора с ровно этой аудиторией — стена.

### Что при этом можно

- **Стрим Claude Code доступен.** `claude -p --output-format stream-json` отдаёт NDJSON
  (`system/init`, вызовы/результаты инструментов, `api_retry`, `result`); с
  `--include-partial-messages` — токенные дельты. Документировано.
- **Agent SDK (TS) спавнит тот же CLI** и говорит с ним JSON по stdio. «SDK или сырой stream-json» —
  тактика; оба упираются в авторизацию выше.
- **Дослать ход в запущенный `claude -p` нельзя** (одна инвокация — один промпт, дальше
  `--resume`). **К чужой интерактивной сессии наблюдателем не подключиться.** Транскрипты в
  `~/.claude/projects/**/*.jsonl` — формат внутренний, меняется между релизами, не парсить.
- **MCP наружу и хуки документированы и поддерживаются** — наша легальная дверь (см. C).

### Синтаксис: TextMate, не TreeSitter

Грамматики взяты через `vscode-textmate` + `vscode-oniguruma` — **та же сделка, что везде:
техническая современность в обмен на выполнимость соло**. TreeSitter объективно лучше (инкрементальное
дерево, структурные запросы/выделения, настоящий фолдинг), но у него **свой** мир грамматик, который
надо собирать и содержать — а это `languages.toml` Helix на 195 человек либо деньги Zed. TextMate
даёт 47 языков даром. И это **не компромисс изгоя — это архитектура VS Code**: TextMate для мгновенной
первой отрисовки + semantic tokens от LSP для точности. Цена: онигуруму на большом файле надо гнать
async/по видимому окну (см. `SyntaxHighlighting` в TODO), а фолдинг пока indentation-based. Публичный
ответ на «почему не TreeSitter» вести **экосистемой** («так я получаю грамматики VS Code целиком и
бесплатно, и так делает сам VS Code»), не удобством.

## Принципы

### Батарейки вшиваем, а не раздаём

«Легко настраивается» и «запускает расширения VS Code» тянут в разные стороны. Скажешь «у нас
расширения VS Code» — люди пойдут их ставить, упрутся в OpenVSX и получат ровно тот опыт настройки,
от которого бежали. Extension host **соблазняет в ловушку Neovim**: в секунду, когда даёшь точку
расширения наружу, ты вернул пользователю конфиг-проект. Helix пять лет держался тем, что расширять
было **нечем**; нам ту же дисциплину придётся держать волей, а не отсутствием механизма.

Значит: **не «иди поставь расширение для Python», а курируемый набор языковых серверов, который
просто есть после установки.**

### Что говорить вовне

**Не «запускает расширения VS Code»** (обещание, сдерживаемое лишь частично).
**А «языки просто работают»**: единственный терминальный редактор, где Python и TypeScript работают
из коробки и не надо учить модальность. Но это заголовок про **порог**, а не про причину. Причина —
**«VS Code кончается на ssh, а тут не кончается».**

## Анти-цели

- **Не позиционируемся как клон VS Code.** В этой рамке мы всегда хуже; тред заканчивается «а зачем,
  есть Remote-SSH / code-server». Мы не против VS Code — мы там, где его нет.
- **Не гонимся за паритетом фич.**
- **Не монетизируем редактор.** Ни лицензия, ни open core, ни pro-фичи.
- **Не закрываем исходники.** Продавать не собираемся → закрытость ничего не защищает, только
  вычитает: убивает и adoption, и карьерный рычаг (за бинарник не нанимают; нанимают за читаемый
  код). Терминальная аудитория закрытый редактор не примет в принципе. Страх «утащат» → копилефт, не
  закрытость.
- **Не реверсим закрытые протоколы.** `ide` MCP Claude Code — мимо.
- **Не строим стратегию на своей трактовке чужого ToS.**
- **Появление №1 — про продукт, не про то, как он создан.** Метод (как агент довёл 72k строк до
  связного состояния) силён, но как **заголовок** ставит в позу защиты («докажи, что не слоп»).
  Продукт как заголовок ставит в позу демонстрации. Метод придерживаем — он берёт взятку **после**
  того, как продукт вызвал уважение, и живёт в заготовках ответов, не в посте (см. ниже).

## Лицензия

У Zed полезно подсмотреть **сплит лицензий**: редактор под GPL-3.0 (копилефт, защита от
проприетарных форков), а **GPUI — UI-фреймворк — под Apache-2.0**, пермиссивно, чтобы брали.
Компонент — пермиссивно, продукт — копилефт. Для нас перекос ещё сильнее в пермиссивность: **TUIDom
и extension host — MIT/Apache-2.0**, потому что весь смысл направления C в том, чтобы их **взяли**, в
том числе компании; AGPL/GPL на инфраструктуре убивает ровно то усыновление, ради которого всё
затевается.

## Направления

Не взаимоисключающие — разная цена и разный потолок.

### A. Немодальный редактор с батарейками ⭐ — вход
Редактор, где языки работают из коробки, настройка не нужна, модальность учить не надо. В консоли, не
вим. Аудитория — вытесненные в терминал (см. выше) + классический reach (ssh, контейнеры, залоченные
окружения). Это единственное, чего можно **захотеть** на первый день, то есть единственный вход для
adoption. Не блокируется фазами Extensions — блокируется качеством дефолтов LSP. Роль: вход, всё
остальное — следствие.

### B. Агентный редактор — витрина
Агент правит код, ты видишь правки инлайн и правишь дифф руками до апрува — в терминале, по ssh, без
GUI. Не встраивать агента (стена авторизации + гонка за UX харнесса): пользователь запускает **свой**
`claude` со своей подпиской, интеграция — через документированные поверхности (C). Роль: демо-кадр.

### C. Extension host как инфраструктура — глубина и защищаемость
Vexx отдаёт наружу **MCP-сервер**, дающий любому агенту настоящие языковые фичи VS Code
(go-to-definition, find-all-references, rename-symbol, code actions, диагностики) вместо grep'а. Плюс
`PreToolUse`-хуки: перехват Edit/Write → правка в наш diff-вьювер. Авторизация — пользователя, ноль
второго чека. Функционально то же, что официальный `ide`, но легально и без реверса; сверх того —
весь языковой мозг VS Code, которого у `ide` нет. Работает с любым агентом, предъявляется отдельно от
редактора. Минус: сторонний MCP-сервер вызывается **по решению модели** — привилегий `ide` не будет,
хуки закрывают часть разрыва. **Не зависит от качества редактора** — самостоятельный артефакт.

### D. TUI-движок как компонент — побочный эффект
Технически лучше Ink и в том же языке, но рынок крошечный, Ink бесплатен, переезд = переписывание UI.
Роль: побочный эффект, не направление.

### E. Карьерный рычаг — деньги
Проект как proof-of-work: 72k строк, движок рендера с нуля, extension host на RPC, LSP — соло за
месяцы. Внимание конвертируется не в выручку, а в позицию. **Самый реалистичный способ «чтобы платили
за работу над редактором»** — как зарплата, не revenue. Кандидаты: Coder, Gitpod, Warp, Replit,
Sourcegraph, Zed, Anthropic/OpenAI/Cursor. Дополнительный, придерживаемый козырь — **метод** (как
получать от агентов большой связный код на дистанции месяцев): то, чем сейчас мучается каждый
инженерный отдел, и у автора не мнение, а работающий конвейер + 72k строк доказательства. Рассказывать
потом, не в первом появлении.

## Текущая ставка

**A — вход. C — глубина. B — витрина. E — деньги. D — побочный эффект.**

A даёт людей, C даёт защищаемость, обе выплаты идут с одного актива — extension host'а. Обёртки живут
до следующего релиза платформы; экосистему расширений VS Code без Electron Anthropic не построит
никогда — это не их бизнес. Это наша постоянная территория.

## План выхода на публику

**Два артефакта — два появления, разные публики, ни один не жжёт другой.**

### Появление №1 — сейчас: эссе про продукт, не Show HN

Инстинкт «не с полноценным супер-редактором, а с идеей и претензией» — верный. **Show HN** значит
«попробуй прямо сейчас» и требует полировки; **эссе** значит «прочитай» и требует идеи с
доказательством. Show HN у нас один — не тратить его на сырой редактор.

- **Претензия:** экосистема расширений VS Code не обязана жить в Electron; терминал может получить
  настоящую IDE, а не редактор с подсветкой. На первом плане — **extension host без Electron**, не
  «удобный редактор» (публика HN ≠ наш пользователь: ей нужен технический подвиг, и это же
  обслуживает E — нанимающие сидят там).
- **Доказательство:** скриншот. Терминал, выглядящий как VS Code — тема Dark+, TextMate-грамматики,
  живой LSP, автокомплит. Реакция «стоп, это терминал?» — за полсекунды, без веры на слово.
- **Что должно быть правдой до:** старт измерен и не стыдный (✅ 26мс над полом Node); ставится одной
  командой (SEA → `curl | sh`/brew); перф-таблица против nvim+config снята своей рукой; ответы на
  неизбежные комментарии написаны заранее (не neovim/helix, не Remote-SSH, Node/перф, webview'ы,
  marketplace/легальность, **и «это вайбкодинг?» — короткий спокойный ответ про метод, в
  комментариях, если спросят, не в посте**); потолок назван вслух («языковые и тулинговые расширения
  — да, webview — никогда»).
- **Чего в посте быть не должно:** слова «клон VS Code»; обещания агентной истории раньше, чем она
  работает; превосходства над VS Code в любой форме; метода как заголовка.

### Появление №2 — позже: MCP-сервер, отдельно
«Твой агент перестаёт грепать» — другая публика (AI-CLI, компании), другая польза (прямая дорога к
E). Требует LSP (есть) + хуки. Не блокируется ни маркетплейсом, ни редактором.

### Что НЕ блокирует появление №1
PieceTree · Extensions Phase 9/10 (`.vsix`, реестр) · агентная интеграция · паритет с чем угодно.

## Демо, ради которого всё

> Агент правит код по ssh в контейнере без GUI. Правки прилетают в редактор через `PreToolUse`-хук и
> ложатся инлайн. Ты правишь дифф руками до апрува. Агент делает rename через **настоящий language
> server**, а не регекспом — потому что мы дали ему свой extension host как MCP. Пользователь платит
> только за свою подписку.

Ни VS Code (нужен GUI), ни Claude Code в голом терминале (нет редактора и языкового мозга), ни Neovim
(нет extension host'а) этого не показывают. Релиз привязываем к этому кадру, не к паритету фич.

## Риски

- Часть HN скажет «just use neovim» / «AI-слоп» до чтения — переживаемо, если ответы готовы.
- Окно закроется **не Хеликсом, а если Claude Code отрастит сносный редактор диффов** и схлопнет нишу
  сверху. Против этого работает только extension host, который они строить не станут.
- Пост даёт **внимание, а не пользователей.** Пользователи придут потом и по другой причине — потому
  что их вытеснили. Таймскейл Helix — годы, не недели; соло-проект на дистанции побеждает не
  скоростью, а тем, что не умирает.
