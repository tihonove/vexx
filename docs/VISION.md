# Vision — позиционирование Vexx

Этот документ — не про архитектуру и не про фичи. Он про то, **чем Vexx является для внешнего
мира** и куда осмысленно двигаться. Цели и неизменяемые ограничения реализации — в
[GOAL.md](../GOAL.md); здесь — стратегия, которая пересматривается по мере того, как меняются
факты.

Документ живой. Если факт из раздела «Ограничения реальности» перестал быть фактом — направление
пересматривается.

---

## Активы

- **VS Code-совместимый extension host вне Electron и браузера.** Расширения VS Code, запущенные
  там, где нет ни Electron, ни браузера. Аналогов, по сути, нет.
- **TUI-движок на TypeScript, который не Ink.** Grid + double buffering + диффы ANSI + layout +
  Kitty keyboard protocol.
- **LSP.** Работает — значит это IDE, а не «редактор с подсветкой».
- **Собственно редактор** — немодальный, с мышечной памятью VS Code.

**Ранжировать их — ошибка.** Редактор — слабейший актив как вещь, которую *продают*, и сильнейший
как вещь, которую *принимают*: захотеть можно только редактор, никто не просыпается с желанием
поставить MCP-сервер. А extension host — не отдельная ставка, а **механизм**, которым редактор
получает батарейки: языковая поддержка VS Code живёт в расширениях, мы их запускаем — значит языки
у нас есть даром.

Один актив, две выплаты: **для человека — языки из коробки, для агента — языковой мозг через MCP.**

---

## Аудитория — кто именно приходит

Терминал растёт не сам по себе. Он растёт **потому что туда загнали агенты**: Ghostty, Warp, tmux,
девконтейнеры — фон, а двигатель — Claude Code и его родня.

Ключевое: аудитория, которая сейчас прибывает в терминал, — это **не терминальные люди**.
Терминальные люди никуда не уходили и давно поделены вимом. Прибывают вэсэкодовцы, которые раньше
заходили в терминал за `git push`, а теперь сидят там часами, потому что там живёт агент. Они по
определению не вимеры, у них мышечная память VS Code, и редактировать им всё равно надо.

Популяция новая, растущая, и её не обслуживает никто.

**Наша ниша — не «терминальный редактор»** (статично, занято вимом), **а «редактор для тех, кого
агенты затащили в терминал»**.

## Конкурентная дыра

Терминальные редакторы делятся на две кучки, и между ними пусто.

| | Модальность | Языки из коробки | Настройка |
|---|---|---|---|
| Neovim | модальный | нет | отдельный проект (отсюда NVChad, LazyVim, AstroNvim) |
| **Helix** | **модальный** (свои биндинги в стиле kakoune — переучиваться даже вимеру) | **да** | не нужна |
| micro | немодальный | почти нет | не нужна |
| MS `edit` | немодальный | нет (демонстративно минималистичен) | не нужна |
| **Vexx** | **немодальный, VS Code** | **да** | **не нужна** |

**Немодальность + языки из коробки + мышечная память VS Code + ноль конфига — не занято никем.**

- **Главный конкурент — Helix, а не Neovim.** Он честно решает ту же боль («LSP из коробки, конфиг
  не нужен»). Наша дифференциация против него — одно слово: **немодальный**.
- Против micro и `edit` — языки.
- То, что рынок породил NVChad и LazyVim, — признание, что «настройка Neovim» это отдельный проект.

## Ограничения реальности

Потолки, которые нам не сдвинуть. Стратегия обязана их обходить, а не бороться с ними.

- **Marketplace VS Code закрыт юридически.** ToS Microsoft запрещает сторонним продуктам ходить в
  marketplace — отсюда план с GitHub-топиком и OpenVSX (см. [TODO/Extensions.md](TODO/Extensions.md),
  Phase 10). Следствие: на OpenVSX нет ровно того, ради чего терпят VS Code — Pylance, C#, Copilot,
  Remote-SSH. Они лицензированы только для VS Code.
- **Webview'ы в TUI невозможны в принципе.** Потолок совместимости — языковые и тулинговые
  расширения, не UI-heavy.
- **Функциональный паритет с VS Code недостижим.** Это не горизонт, а асимптота, часть которой
  закрыта by design. Строить позиционирование на паритете нельзя.
- **Редакторы не монетизируются.** VS Code бесплатен, потому что Microsoft продаёт Azure и GitHub.
  Neovim и Helix бесплатны. Даже Zed **не продаёт редактор** — продаёт сервисы и AI (Pro $10/мес —
  это хостед-кредиты к моделям). Продавать Vexx нельзя ни в каком виде.
- **`ide`-протокол Claude Code закрыт.** Официальные интеграции VS Code и JetBrains работают через
  встроенный MCP-сервер `ide`: WebSocket на loopback, случайный порт, токен в
  `~/.claude/ide/<port>.lock`. Протокол **не документирован и не открыт третьим сторонам**. Реверс
  возможен, но это внутренний RPC — ломается на любом релизе.
- **Авторизация claude.ai запрещена третьим лицам.** Дословно из доков Agent SDK: *«Unless
  previously approved, Anthropic does not allow third party developers to offer claude.ai login or
  rate limits for their products, including agents built on the Claude Agent SDK.»* Значит
  **встроенный в Vexx агент потребует отдельный `ANTHROPIC_API_KEY` и отдельный чек** — при том что
  у пользователя уже есть Max. Для бесплатного редактора с ровно этой аудиторией — стена.

### Что при этом можно

- **Стрим Claude Code доступен.** `claude -p --output-format stream-json` отдаёт NDJSON:
  `system/init`, вызовы и результаты инструментов, `api_retry`, финальный `result`. С
  `--include-partial-messages` — токенные дельты. Документировано.
- **Agent SDK (TS) спавнит тот же CLI** и говорит с ним JSON по stdio — не переписывает харнесс и не
  ходит в API напрямую. «SDK или сырой stream-json» — выбор тактический. Оба упираются в
  авторизацию выше.
- **Дослать ход в уже запущенный `claude -p` нельзя** (одна инвокация — один промпт, дальше
  `--resume <session-id>`). **Подключиться наблюдателем к чужой интерактивной сессии нечем.**
  Транскрипты в `~/.claude/projects/**/*.jsonl` лежат, но доки предупреждают: формат внутренний,
  меняется между релизами, не парсить.
- **MCP наружу и хуки — документированы и поддерживаются.** Claude Code штатно ходит в сторонние
  MCP-серверы; `PreToolUse`-хуки штатно перехватывают вызовы инструментов. Это наша легальная дверь.

## Принципы

### Батарейки вшиваем, а не раздаём

«Легко настраивается» и «запускает расширения VS Code» тянут в разные стороны. Скажешь «у нас
расширения VS Code» — люди пойдут их ставить, упрутся в OpenVSX (Pylance нет, C# нет) и получат
ровно тот опыт настройки, от которого бежали.

Значит: **не «иди поставь расширение для Python», а курируемый набор языковых серверов, который
просто есть после установки.** Проще в реализации, честнее, и ровно то, чего хочет пользователь.

### Что говорить вовне

**Не «запускает расширения VS Code»** — обещание, которое мы можем сдержать лишь частично.
**А «языки просто работают»**: единственный терминальный редактор, где Python и TypeScript работают
из коробки и при этом не надо учить модальность. Extension host — деталь реализации, благодаря
которой это правда.

Понятно за секунду, бьёт в реальную боль, не приглашает сравнение с VS Code.

## Анти-цели

- **Не позиционируемся как клон VS Code.** В этой рамке мы всегда хуже: тред заканчивается «а зачем,
  есть Remote-SSH / code-server». Клоны получают не внимание, а сравнения.
- **Не гонимся за паритетом фич.**
- **Не монетизируем редактор.** Ни лицензия, ни open core, ни pro-фичи.
- **Не закрываем исходники.** Цель — присоседиться, а это видимость и adoption; закрытость убивает
  оба. Продавать мы всё равно не собираемся, значит закрытость не защищает ничего. Терминальная
  аудитория закрытый редактор не примет в принципе, а карьерный рычаг (E) — это **читаемый код**:
  за бинарник не нанимают. Если появится страх, что «утащат», ответ — копилефт, а не закрытость.
- **Не реверсим закрытые протоколы.** `ide` MCP Claude Code — мимо: фрагильно, неподдерживаемо, и
  ставит нас в позу самозванца вместо соседа.
- **Не строим стратегию на своей трактовке чужого ToS.**

## Лицензия

У Zed полезно подсмотреть **сплит лицензий**: редактор под GPL-3.0 (копилефт, защита от
проприетарных форков), а **GPUI — UI-фреймворк — под Apache-2.0**, пермиссивно, чтобы его брали.
Компонент — пермиссивно, продукт — копилефт.

Для нас логика та же, но с перекосом в пермиссивность: **TUIDom и extension host должны быть
максимально пермиссивны (MIT/Apache-2.0)**, потому что весь смысл направления C — чтобы их
**взяли**, в том числе компании. AGPL/GPL на инфраструктуре убивает ровно то корпоративное
усыновление, ради которого всё затевается, а усыновление компанией — это и есть путь к E.

## Направления

Не взаимоисключающие — но у них разная цена и разный потолок.

### A. Немодальный редактор с батарейками ⭐

**Тезис:** редактор, где языки работают из коробки, настройка не нужна, и не надо учить модальность.
В консоли. Не вим.

**Аудитория:** те, кого агенты затащили в терминал (см. выше), плюс классический reach — SSH,
bastion-хосты, контейнеры, tmux, узкий канал, залоченные окружения без браузера.

**За:** дыра реально пуста (см. таблицу). Это единственное, чего можно **захотеть** на первый день,
то есть единственный вход для adoption. Скрёбёт собственный зуд автора — лучший предиктор хорошего
dev-tool'а.
**Против:** эта аудитория не платит. Потолок в деньгах нулевой — но деньги мы и не берём.
**Роль:** вход. Всё остальное — следствие.

### B. Агентный редактор — человек и агент в одном буфере

**Тезис:** агент правит код, ты видишь правки инлайн и правишь дифф руками до апрува — в терминале,
по SSH, без GUI.

**За:** опыт VS Code + Copilot, которого в терминале нет ни у кого. Отличный демо-кадр.
**Против:** через встраивание Agent SDK — стена авторизации (второй чек) плюс переизобретение UX
харнесса и гонка за их скоростью.
**Как обойти:** не встраивать агента. Пользователь запускает **свой** `claude` со своей подпиской, а
интеграция идёт через документированные поверхности (см. C).
**Роль:** витрина.

### C. Extension host как инфраструктура — языковой мозг для любого агента

**Тезис:** Vexx отдаёт наружу MCP-сервер, дающий **любому** агенту настоящие языковые фичи VS Code —
go-to-definition, find-all-references, rename-symbol, code actions, диагностики — вместо grep'а.

**Механика (всё из легальных документированных кубиков):**
- **MCP-сервер поверх extension host + LSP (pull).** Инструменты: языковая интеллектуальность,
  активный файл, выделение, открыть дифф.
- **`PreToolUse`-хуки (push).** Перехват Edit/Write → правка едет в наш diff-вьювер.
- Авторизация — пользователя, его подписка, его `claude`. Ноль ToS-проблем, ноль второго чека.

**За:** функционально то же, что официальный `ide`-сервер, но легально и без реверса. Плюс то, чего
у официального нет: их `ide` отдаёт диагностики и запускает питон в ноутбуке, наш отдаёт весь
языковой мозг VS Code. Работает с любым агентом. Предъявляется **отдельно от редактора**.
**Против:** сторонний MCP-сервер вызывается **по решению модели** — привилегий официального `ide` не
будет. Хуки закрывают часть разрыва, но не весь.
**Роль:** глубина и защищаемость.

### D. TUI-движок как компонент

**За:** технически мы лучше Ink, и в том же языке.
**Против:** рынок крошечный, Ink бесплатен, переезд = переписывание UI. Claude Code закрыт, Gemini
CLI на Ink, opencode/Crush на Go.
**Роль:** побочный эффект.

### E. Карьерный рычаг

**Тезис:** проект как proof-of-work. 72k строк, движок рендера с нуля, extension host на RPC, LSP —
соло за месяцы. Внимание конвертируется не в выручку, а в позицию.

**За:** **самый реалистичный способ «чтобы платили за работу над редактором»** — как зарплата, а не
как revenue. Кандидаты: Coder, Gitpod, Warp, Replit, Sourcegraph, Zed, Anthropic/OpenAI/Cursor.
**Против:** не контролируется напрямую, зависит от момента.
**Роль:** способ получить деньги.

## Текущая ставка

**A — вход. C — глубина. B — витрина. E — деньги. D — побочный эффект.**

Логика связки: **A даёт людей, C даёт защищаемость, и обе выплаты идут с одного актива** — extension
host'а. Для человека он батарейки, для агента языковой мозг.

Обёртки живут до следующего релиза платформы. Экосистему расширений VS Code без Electron Anthropic
не построит никогда — это не их бизнес. Это наша постоянная территория.

---

## Демо, ради которого всё

Один кадр, который не показывает никто:

> Агент правит код по SSH в контейнере без GUI. Правки прилетают в редактор через `PreToolUse`-хук
> и ложатся инлайн. Ты правишь дифф руками до апрува. Агент делает rename через **настоящий language
> server**, а не регекспом — потому что мы дали ему свой extension host как MCP. Пользователь платит
> только за свою подписку.

Ни VS Code (нужен GUI), ни Claude Code в голом терминале (нет редактора и нет языкового мозга), ни
Neovim (нет extension host'а) этого не показывают.

**Релиз привязываем к этому кадру, а не к паритету фич.**

## Открытые вопросы

- Где живёт MCP-сервер относительно слоёв (Extensions / Controllers)? Какой минимальный набор
  инструментов делает демо неотразимым?
- Какой курируемый набор языковых серверов вшиваем в дефолт, чтобы «языки просто работают» было
  правдой в день релиза? Чем закрываем дыры OpenVSX (Python без Pylance, C#)?
- Насколько далеко хуки закрывают разрыв с привилегиями официального `ide`-сервера?
- Стоит ли держать встроенного агента как опцию для тех, у кого есть API-ключ, — или это распыление?
- Публичная формулировка потолка совместимости («языковые и тулинговые расширения — да, webview —
  никогда») как позиция: это сила, а не слабость.
